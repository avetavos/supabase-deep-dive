## คอร์สนี้ทำงานยังไง

<CardGrid>
  <Card title="เป็น Postgres บวกของเพิ่ม" icon="rocket">
    ทุกหัวข้อเริ่มจาก Postgres ที่อยู่ข้างใต้ — auto-generated API, Realtime และ Storage ล้วนเป็น layer บน database จริงตัวเดียว สิ่งที่คุณเรียนจึงใช้กับ Postgres ธรรมดาได้ด้วย
  </Card>
  <Card title="default ปัจจุบัน ที่ verify แล้ว" icon="random">
    เขียนตามพฤติกรรมปัจจุบันของ Supabase และตรวจสอบกับ official docs ผ่าน context7 — รวมถึง Data API exposure model ที่ต้อง grant ชัดเจน และการเปิด Realtime ต่อ table ที่ tutorial เก่าหลายที่ข้ามไป
  </Card>
  <Card title="security สองชั้น ไม่ใช่ชั้นเดียว" icon="approve-check">
    ไม่ใช่แค่ "เปิด RLS" แต่เป็นภาพรวมทั้งหมด — Data API เปิด table ไหนให้เข้าถึงได้บ้าง และ Row Level Security จำกัด row ไหนที่เห็นได้บ้าง สองคำถามที่แยกกันและมีคำตอบต่างกัน
  </Card>
  <Card title="ไดอะแกรมที่อธิบายได้" icon="puzzle">
    สถาปัตยกรรมของ Supabase, เส้นทางตัดสินใจของ RLS, publication model ของ Realtime และโหมด pooling ของ Supavisor เป็นไดอะแกรม Mermaid ที่รองรับธีม สองภาษา (English / ไทย) พร้อมบันทึกความคืบหน้า quiz
  </Card>
</CardGrid>
