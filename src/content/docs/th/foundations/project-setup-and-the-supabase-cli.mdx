## ไอเดียในหนึ่งประโยค

Supabase project เริ่มต้นจาก Postgres instance แบบ hosted พร้อม API key สองตัวที่ระดับความน่าเชื่อถือต่างกันมาก และ Supabase CLI คือสิ่งที่ทำให้คุณ link folder บนเครื่องเข้ากับ project นั้น แล้วรันทั้ง stack บนเครื่องตัวเองด้วย Docker ได้

## สร้าง project และ key สองตัวของมัน

คุณสร้าง hosted project ใหม่ได้จาก dashboard ของ Supabase หรือ shortcut `database.new` เบื้องหลังมันจะ provision Postgres instance ของคุณเอง, API URL และ API key สองตัว

- **`anon` key** — public ใช้ใน client-side code ได้ปลอดภัย (browser bundle, mobile app) request ที่ใช้ key นี้ต้องผ่าน Row Level Security
- **`service_role` key** — secret, full-access, **bypass Row Level Security ทั้งหมด** ใช้เฉพาะกับ server-side code ที่เชื่อถือได้เท่านั้น

<Callout title="ห้าม expose service_role key ใน client-side code เด็ดขาด">
`service_role` key ข้าม RLS policy ทุกอันในทุก table ถ้ามันหลุดไปอยู่ใน browser bundle, mobile app หรือ code ที่ user เปิดดูได้ ใครก็ตามที่มี key นี้อ่านและเขียน database ทั้งหมดของคุณได้เลย เก็บมันไว้ใน environment variable ฝั่ง server เท่านั้น ห้ามอยู่ใน code ที่ ship ไปหา client เด็ดขาด นี่คือกฎที่คอร์สนี้จะย้ำอีกหลายครั้ง
</Callout>

```typescript
// Server-side only — e.g. an API route, a cron job, an Edge Function.
// process.env.SUPABASE_SERVICE_ROLE_KEY must never be bundled into client code.
import { createClient } from '@supabase/supabase-js';

const supabaseAdmin = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
);
```

## Supabase CLI

CLI คือวิธีจัดการ project เป็น code แทนที่จะทำผ่าน dashboard อย่างเดียว

```bash
# Scaffold a local supabase/ config directory in your project
npx supabase init

# Authenticate the CLI with your Supabase account
supabase login

# Connect this local project folder to a specific hosted project
supabase link --project-ref <project-ref>
```

`supabase link` คือคำสั่งที่บอกว่า folder `supabase/` บนเครื่องคุณตรงกับ hosted project ตัวไหน — `db push` หรือ `db diff` ครั้งต่อ ๆ ไปจะเล็งไปที่ project ที่ link ไว้นี้เสมอ

## Local development ด้วย `supabase start`

```bash
supabase start
```

คำสั่งนี้จะสตาร์ท **Supabase stack ทั้งชุดบนเครื่องตัวเองผ่าน Docker** — Postgres, Studio, Auth, Storage, Realtime และอื่น ๆ รันเป็น container บนเครื่องคุณทั้งหมด นั่นแปลว่าคุณ develop และ test schema, RLS policy และ client code ได้แบบ offline เต็มรูปแบบ โดยไม่มีความเสี่ยงต่อ hosted project เลย ก่อนที่จะรัน `supabase db push` ใส่มันจริง ๆ

Hosted project ให้ Postgres มาพร้อม API URL, key **`anon`** สำหรับ client code (ต้องผ่าน RLS) และ key **`service_role`** สำหรับ server code ที่เชื่อถือได้ (bypass RLS — ห้าม ship ไปหา client เด็ดขาด) คำสั่ง CLI อย่าง `supabase init`, `supabase login` และ `supabase link --project-ref` เชื่อม local folder เข้ากับ project นั้น ส่วน `supabase start` รันทั้ง stack บนเครื่องตัวเองผ่าน Docker ทำให้ develop แบบ offline ได้เต็มรูปแบบ
</Callout>
